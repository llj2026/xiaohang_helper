小航AI助手 —— 系统设计文档
 
一、总体架构
本系统基于Python + Streamlit开发前端交互界面，调用大模型远程API，读取本地Markdown知识库实现校园智能问答。
整体分为四层：
1.交互层（Streamlit页面）：身份选择、输入框、推荐问题标签、对话展示、历史记录
2.业务逻辑层：知识库加载、提示词拼接、会话状态管理、扩展功能控制
3.接口调用层：封装HTTP请求，向大模型API发送请求并解析返回结果
4.数据层： data/ 目录下Markdown知识库文件
程序工作流程：用户选择身份→输入问题→程序自动加载全部本地知识库→拼接系统提示词+知识库文本+用户问题→调用API→获取AI回复→页面展示回答、元信息、保存对话历史。

三、模块详细设计
模块1：知识库加载模块
文件位置： app.py 
功能：自动扫描 data/*.md 所有Markdown文件，读取全部文本内容，合并作为参考资料送入大模型
核心代码逻辑：使用 Path("data").glob("*.md") 通配读取，新增md文件无需修改代码
输入：无
输出：合并后的知识库完整字符串
模块2：提示词（Prompt）构造模块 
功能：根据用户选择身份（新生/在校生/教师）加载对应角色设定，拼接约束规则、知识库内容、用户问题
约束规则：禁止编造信息，知识库无内容统一回复无法解答；只回答郑州航空工业管理学院相关问题
输出：构造完成的JSON请求消息体
模块3：API请求模块
功能：向硅基流动API发送POST请求，增加超时控制
扩展：加入  st.spinner  加载动画，请求期间显示“小航正在思考...”
输入：构造好的对话消息
输出：AI原始回答文本、接口耗时
模块4：Streamlit界面交互模块
1.身份选择组件：单选框切换用户身份
2.问题输入组件：文本输入框提交问题
3.分类标签页（扩展功能）
三个标签：新生指南、办事流程、应急防骗；内置快捷问题按钮，点击自动填充输入框
4.对话展示区：展示AI回复，下方显示元信息（回答字数、接口耗时）
5.会话历史模块（扩展功能）
使用  st.session_state["history"]  存储每条记录：时间、身份、问题、回答；按倒序展示最新对话
6.清空历史按钮（扩展功能）
点击清空history列表并刷新页面
模块5：会话状态管理
依靠  st.session_state  实现：
-保存当前选中用户身份
-持久保存全部问答历史记录
-记录待提交快捷问题

四、核心数据流
用户操作 → 界面组件采集信息
→ 加载全部知识库
→ 拼接角色Prompt + 知识库 + 用户提问
→ API网络请求（显示加载动画）
→ 接收AI回答
→ 计算字数、接口耗时
→ 页面渲染回答与元信息
→ 将本条对话存入会话历史

五、扩展功能设计详情
1.会话历史记录
存储字段：时间 time 、身份 role 、问题 question 、回答 answer ；页面底部倒序展示对话。
2.清空对话按钮
清除session内history数组，执行页面刷新。
3.加载状态提示 spinner
API请求上下文包裹，请求完成自动消失。
4.问题分类标签 tabs
分组放置常见咨询按钮，点击赋值问题变量触发页面运行。
5.回答元信息展示 caption
统计文本字符数量、计算接口耗时，小字展示在回答下方。

六、接口设计
大模型API接口
请求方式：POST
请求头：鉴权密钥、内容类型
请求体：包含模型名称、消息列表（系统提示词+用户消息）
返回：JSON格式，提取路径  choices[0]["message"]["content"] 

七、非功能需求设计
1.容错处理
检测空输入，禁止空白问题发起请求；网络超时友好提示。
2.版本管理规范
每个功能开发完成执行一次Git提交，规范提交注释示例：
git add src/
git commit -m "feat: 增加会话历史记录功能"
git push
3.可维护性
新增知识库文件直接放入data文件夹，无需修改代码；功能模块化，便于增删扩展功能。
 
八、系统约束
1.AI仅允许使用本地知识库信息作答；
2.知识库无相关内容，禁止编造，统一标准化回复；
3.只处理郑州航空工业管理学院校园相关咨询；
4.严格区分三种用户身份，差异化输出回答内容。
 
九、测试要点
1.切换三种身份提问，验证回答视角存在差异
2.提问 05_交通出行.md 内相关内容，验证新增知识库文件可正常读取
3.测试全部扩展功能：历史记录、清空按钮、加载动画、标签快捷问题
4.测试知识库不存在的问题，检验防幻觉机制是否生效
5.边界测试：空白输入、无关校外话题测试拦截效果
